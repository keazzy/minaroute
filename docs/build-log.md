# Minaroute — Build Log

A shared running log between the **build agent** (Claude Code, in the repo) and the **planning/design partner** (Cowork). Both read and write the same folder, so this file — plus the roadmap — is how progress, decisions, and deviations stay in sync while building. Keep it current, especially for UI changes and anything that deviates from the docs, so the spec doesn't drift.

## How to use this file

- After each task (or any meaningful change), add a dated entry under **Log**.
- Record decisions and deviations under **Decisions & deviations**. If a change is real (not a tiny tweak), also update the relevant doc — `design.md`, `screens.md`, or `prd.md` — so future work builds from the current map.
- Keep `docs/product-roadmap.md` checkboxes and the status line current — that's the canonical progress tracker; this log is the narrative around it.
- Note anything that blocks you, or that the planning partner should weigh in on, under **Open questions / blockers**.

## Decisions & deviations

- **Config lives in `app.json`, not `app.config.ts`.** The roadmap/PRD reference `app.config.ts`; the real repo has always used `app.json`. Kept `app.json` (do-not-re-scaffold). TASK-002 edits applied there.
- **Audio: `expo-audio`, not `expo-av`.** SDK 55 deprecates/removes `expo-av`; `expo-audio` is the supported replacement and is explicitly sanctioned by PRD § Stack Integration. Phase 2's audio player (TASK-024) will target `expo-audio`.
- **Local store: `expo-sqlite`, MMKV omitted.** TASK-007 allows "expo-sqlite (or MMKV)"; SQLite chosen. `react-native-mmkv` not installed to avoid native build risk on the RN 0.83 / New-Arch canary. Revisit only if a perf need appears.
- **New pilgrimage-mode code lives under `src/`** (per PRD § Repository Structure and the roadmap's task file paths), imported via the existing `@/*` → repo-root alias (e.g. `@/src/lib/...`). Existing Minaroute code (`app/`, `components/`, `lib/`, `constants/`, `hooks/`) is untouched — the new layer adds, it does not replace.
- **Status-line convention:** the roadmap header counts "N/58". Phase 0 covers TASK-001–012.
