# Minaroute ‚Äî Build Log

A shared running log between the **build agent** (Claude Code, in the repo) and the **planning/design partner** (Cowork). Both read and write the same folder, so this file ‚Äî plus the roadmap ‚Äî is how progress, decisions, and deviations stay in sync while building. ## How to use this file

- After each task (or any meaningful change), add a dated entry under **Log**.
- Record decisions and deviations under **Decisions & deviations**. If a change is real (not a tiny tweak), also update the relevant doc ‚Äî `design.md`, `screens.md`, or `prd.md` ‚Äî so future work builds from the current map.
- Keep `docs/product-roadmap.md` checkboxes and the status line current ‚Äî that's the canonical progress tracker; this log is the narrative around it.
- Note anything that blocks you, or that the planning partner should weigh in on, under **Open questions / blockers**.

## Decisions & deviations

- **Config lives in `app.json`, not `app.config.ts`.** The roadmap/PRD reference `app.config.ts`; the real repo has always used `app.json`. Kept `app.json` (do-not-re-scaffold). TASK-002 edits applied there.
- **Audio: `expo-audio`, not `expo-av`.** SDK 55 deprecates/removes `expo-av`; `expo-audio` is the supported replacement and is explicitly sanctioned by PRD ¬ß Stack Integration. Phase 2's audio player (TASK-024) will target `expo-audio`.
- **Local store: `expo-sqlite`, MMKV omitted.** TASK-007 allows "expo-sqlite (or MMKV)"; SQLite chosen. `react-native-mmkv` not installed to avoid native build risk on the RN 0.83 / New-Arch canary. Revisit only if a perf need appears.
- **New pilgrimage-mode code lives under `src/`** (per PRD ¬ß Repository Structure and the roadmap's task file paths), imported via the existing `@/*` ‚Üí repo-root alias (e.g. `@/src/lib/...`). Existing Minaroute code (`app/`, `components/`, `lib/`, `constants/`, `hooks/`) is untouched ‚Äî the new layer adds, it does not replace.
- **Status-line convention:** the roadmap header counts "N/58". Phase 0 covers TASK-001‚Äì012.
